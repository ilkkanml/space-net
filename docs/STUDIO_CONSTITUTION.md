# S.P.A.C.E. NET Studio Constitution

## Core Identity
- Calm Industrial Sci‑Fi
- Industrial loneliness
- Observation-first automation
- Long-session comfort
- Persistent progression
- NEXUS-centered restoration fantasy

Core fantasy:
Reactivating a collapsed industrial network through careful automation and expansion.

## Non-Negotiable Principles
- Success is never sold. Appearance is sold.
- Atmosphere > feature quantity
- Systems first, content second
- No player-facing reset
- No pay-to-win
- No FOMO-driven design
- UI must never dominate factory observation
- E.V.A. remains calm, rare, and meaningful
- Controlled additive expansion only

## Lean Production Pipeline
1. Director target
2. Builder additive patch
3. User playtest
4. QA regression review
5. Documentation sync
6. Director lock

## Current Structure
- Director
- Frontend Builder

## Active Direction
- Passive progression foundation stabilized
- Compatibility Gate active
- Milestone 14 planning: Tier 1 Production Expansion

## Future Decision Filter
Before approving any feature:
1. Does it preserve industrial loneliness?
2. Does it increase player stress?
3. Does it harm readability?
4. Does it create scope drift?
5. Does it support the core automation fantasy?
6. Does it preserve long-session comfort?
7. Does it respect player trust?

## Forbidden Drift
- MMO escalation
- Mobile idle patterns
- Popup spam
- Battle pass pressure
- Reward treadmill
- Over-engineered systems
- Feature flooding
- Dashboard simulator direction
